# Senior AI Engineer 面试评分 Rubric（中文）

总则：按 1-10 分评分。证据不足必须降分并标注低置信度。

## 分数锚点

- 9-10：专家级，能给出可复现方法、关键权衡和线上结果
- 7-8：强，具备独立负责模块/项目能力
- 5-6：中等，概念清楚但实战深度有限
- 3-4：偏弱，缺少关键工程细节
- 1-2：不达标，回答空泛或明显错误

---

## 1) Multimodal Tuning（多模态调优）

看点：
- 训练/微调目标定义（SFT/DPO/蒸馏/对齐）
- 数据配比、采样、清洗
- 评测指标与ablation
- 效果与成本权衡

高分信号：
- 能解释为什么选 LoRA/QLoRA/full fine-tune
- 能给出训练前后指标变化和失败实验
- 了解多模态对齐常见坑（噪声标注、模态偏置）

低分信号：
- 只会说“调参数”
- 无实验设计、无对照组、无指标

---

## 2) Evaluation & Data（评测与数据）

看点：
- 离线评测集构建（覆盖、难例、偏差）
- 在线指标（CTR/成功率/人工满意度）
- 回归测试与版本比较

高分信号：
- 有明确质量门槛和发布门禁
- 能描述数据flywheel与标注策略

低分信号：
- 只凭“感觉变好”
- 缺少闭环

---

## 3) Agent Architecture（Agent架构）

看点：
- 规划器、工具调用、状态管理
- 失败重试、幂等、超时与降级
- 多Agent协作边界

高分信号：
- 给出状态机/工作流设计
- 明确处理工具失败与副作用

低分信号：
- 只有“function calling”名词，无执行策略

---

## 4) Memory Systems（短期/长期记忆，重点）

必须考：
- 短期记忆：会话窗口、摘要压缩、上下文注入策略
- 长期记忆：向量库/结构化profile/episodic log
- 写入策略：触发条件、去重、TTL、冲突解决
- 检索策略：召回+重排+权限过滤
- 效果评估：命中率、答案增益、延迟成本

高分信号：
- 能讲清 memory schema 和 pipeline
- 能处理“脏记忆/过期记忆/冲突记忆”
- 有线上指标与回滚方案

低分信号：
- 只会说“存到向量库就好了”
- 无权限、无生命周期管理

---

## 5) Reliability & Cost（可靠性与成本）

看点：
- SLO/SLA、监控告警、日志与trace
- 速率限制、缓存、批处理、模型路由
- Token成本与延迟优化

高分信号：
- 能给出p95/p99优化思路
- 有明确事故处置与复盘机制

低分信号：
- 无线上观念，仅离线demo经验

---

## 6) Security & Governance（安全与治理）

看点：
- 提示注入、防越权调用
- 工具白名单、参数校验、审计日志
- 敏感数据处理与合规

高分信号：
- 对威胁模型有系统理解
- 能给出防护链路而非单点策略

低分信号：
- 把安全等同于“加个规则”

---

## 7) Communication & Ownership（沟通与Owner意识）

看点：
- 技术表达清晰度
- 跨团队推动与决策能力
- 结果导向与复盘习惯

---

## 结论建议

- 通过：核心技术域多数 >=7，且 Memory + Agent 均不低于6；有真实线上闭环案例
- 待定：有潜力但关键域证据不足，需要补面
- 不通过：关键域明显短板（尤其Memory/Agent/可靠性）
- 证据不足：对话样本过少或质量太低

## Senior 技术追问题库（中文，强技术向）

### A. 多模态调优
1. 你在图文任务中如何处理模态不平衡？请给出采样策略和loss权重设置。
2. 你做过的最佳ablation实验是什么？删掉哪个模块后性能下降最多？
3. 当线上分布偏移（domain shift）发生时，你如何做持续学习且避免灾难性遗忘？
4. 解释一次你在对齐阶段（如DPO/RLHF）遇到reward hacking的案例和修复方式。

### B. Memory（短期/长期）
5. 你提到做过长期记忆，请具体讲写入触发条件、去重规则与TTL策略。
6. 检索命中错误时你怎么定位？如何区分 embedding 召回问题与 reranker 问题？
7. 你会如何设计 memory schema（事实、偏好、任务状态）以支持可审计和可删除（GDPR-like）？
8. 长期记忆污染后如何回滚？你会保留哪些版本快照与审计日志？

### C. Agent系统
9. 工具调用失败率升高时，你的重试策略如何避免副作用重复执行（幂等键设计）？
10. 请画出你Agent的状态机：计划、执行、观察、恢复、终止分别如何触发？
11. 多Agent协作时如何防止循环调用和任务漂移？
12. 你如何做tool selection评测，避免“会调用但调用错工具”？

### D. 线上稳定性与成本
13. 如果要求把p95延迟从3.2s降到1.5s，你会做哪些架构级优化？
14. 成本要降30%同时质量不下降，你会怎么做模型路由与缓存分层？
15. 线上质量突然下降，你的30分钟应急排障路径是什么（按优先级列步骤）？

### E. 安全与治理
16. 面对提示注入攻击，你如何在planner、tool layer、executor三层分别防护？
17. 你如何设计权限系统，确保Agent不会跨租户读取记忆？
18. 你会记录哪些审计字段以满足事后追责和合规要求？

## 面试官使用策略

- 每轮只问1题；若回答无数字，下一轮必须追问指标。
- 每个核心域至少2题，其中 Memory 与 Agent 至少各3题。
- 发现“简历写过但答不出实现细节”时，直接降一个等级并记录风险。
